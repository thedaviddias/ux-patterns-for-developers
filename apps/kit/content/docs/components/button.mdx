Buttons are interactive elements that trigger actions or events when clicked or interacted with. They are one of the most fundamental UI components in any application. For comprehensive UX guidelines and patterns, see the [Button Pattern documentation](https://uxpatterns.dev/patterns/forms/button).

## Accessibility

Buttons are accessible by default with proper keyboard support and ARIA attributes. See the [accessibility guidelines](https://uxpatterns.dev/patterns/forms/button#accessibility) for comprehensive details:

- **Keyboard Navigation**: Buttons can be focused with Tab and activated with Enter or Space
- **Screen Readers**: Use descriptive text or `aria-label` for icon-only buttons
- **Focus Indicators**: Visible focus states for keyboard navigation
- **Disabled State**: Properly announced to assistive technologies

### Best Practices

For detailed UX best practices, see the [Button Pattern best practices](https://uxpatterns.dev/patterns/forms/button#best-practices).

1. **Use semantic HTML**: Always use `<button>` for actions, not `<div>` or `<span>`
2. **Provide labels**: Icon-only buttons must have `aria-label`
3. **Indicate loading**: Use `aria-busy="true"` during async operations
4. **Group related actions**: Use `role="group"` with `aria-label` for button groups
5. **Avoid disabled buttons**: Consider showing why an action is unavailable instead

## API Reference

### Button Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "destructive" \| "outline" \| "secondary" \| "ghost" \| "link"` | `"default"` | The visual variant of the button |
| `size` | `"default" \| "sm" \| "lg" \| "icon"` | `"default"` | The size of the button |
| `asChild` | `boolean` | `false` | Merge props onto child element instead of creating button |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `type` | `"button" \| "submit" \| "reset"` | `"button"` | The HTML button type |
| `className` | `string` | - | Additional CSS classes |

</Callout>
