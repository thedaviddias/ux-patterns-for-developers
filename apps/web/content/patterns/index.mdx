This library documents recurring interface patterns the way developers actually need them: what the pattern is for, when it fits, where it breaks down, and how to implement it responsibly.

## What you will find here

- Pattern pages with anatomy, examples, accessibility notes, and implementation guidance
- Coverage across navigation, forms, feedback, content, data display, authentication, AI, commerce, and more
- Comparisons to adjacent patterns when the choice is not obvious

## How to use the library

Start with a pattern when you already know the interaction you are evaluating. If the decision is still open, move to the [Guides](/pattern-guide) section first and come back once the trade-off is clear.
